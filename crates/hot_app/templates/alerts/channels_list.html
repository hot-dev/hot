About Alert Channels

Alert channels define patterns that match alert events. When an alert is triggered, the system checks which channels match the alert's channel name using regex patterns.

Built-in System Channels

  • run:failed - Triggered when a run fails (after all retries exhausted)
  • run:cancelled - Triggered when a run is cancelled
  • task:failed - Triggered when a task fails or times out
  • task:cancelled - Triggered when a task is cancelled
  • deploy:failed - Triggered when a deployment fails
  • deploy:succeeded - Triggered when a deployment succeeds

Custom Channels

You can create custom channels with regex patterns to match specific alert types. For example, run:.* would match all run-related alerts.
